## 📊 All 6 Equations at a Glance

### Eq. 1 – Newton-Euler (6 equations)
Longitudinal, lateral, vertical, and 3 rotational moment equations

### Eq. 2 – Tire Force Distribution
```
α = S_r / (S_r + S_f)
```
Controls how braking/acceleration is split between rear and front

### Eq. 3 – Roll Angle Kinematics
```
φ = arctan(a_y / g)
```
Roll angle determined directly by lateral acceleration

### Eq. 4 – Vertical Load Transfer
```
N_r = (a' + h' cos φ) M g - (a' + h' a_y/√(1+a_y²)) M g
N_f = (b' - h' cos φ) M g - (b' - h' a_y/√(1+a_y²)) M g
```
Load shifts between front/rear wheels with roll and inertia

### Eq. 5 – Lateral Forces (simple)
```
F_r / N_r = F_f / N_f = a_y / g
```

### Eq. 6 – Longitudinal Forces (complex)
Accounts for load transfer effects on braking/acceleration capacity

model = MotorcycleDynamicsModel()
phi_eq = model.eq_3_roll_angle_kinematics()
phi_solution = phi_eq.subs([(model.a_y, 5.0), (model.g, 9.81)])

## 🧮 All Accessible Symbols

**Geometry:**
- `M` – mass
- `p` – wheelbase
- `a_var` – CoM distance from rear
- `h` – CoM height
- `g` – gravity

**Dynamics:**
- `phi` – roll angle
- `a_x`, `a_y` – longitudinal, lateral acceleration

**Tire Forces:**
- `S_r`, `S_f` – longitudinal (rear, front)
- `F_r`, `F_f` – lateral (rear, front)
- `N_r`, `N_f` – normal (rear, front)

**Non-dimensional:**
- `h_bar` – h/p
- `b_bar` – b'/p
- `b_prime_bar` – (p-b)/p
- `alpha` – force distribution parameter

## ⚡ Performance Cheat Sheet

| Task | Time | Solution |
|------|------|----------|
| Print equations | <1 sec | `model.print_equations()` |
| Single substitution | 0.1–1 sec | Use `.subs()` directly |
| **Repeated evaluation** | **1–10 sec** | **Use `lambdify()`** |
| Solve algebraically | 1–30 sec | Use `sp.solve()` |
| Optimization | 1–10 sec | Use `scipy.optimize` with lambdified functions |

**Golden Rule:** For >10 evaluations, use `lambdify()`!

## 🎓 Key Concepts

**Roll Angle:** Determined purely by lateral acceleration  
**Load Transfer:** Nonlinear due to sqrt(1+a_y²) term  
**Tire Forces:** Limited by friction ellipse (Pacejka model)  
**Feedforward:** S/N ratio depends on CoM position relative to wheel axes

## 💡 Pro Tips

### Tip 1: Use LaTeX output for papers
```python
import sympy as sp
phi = model.eq_3_roll_angle_kinematics()
print(sp.latex(phi))  # → LaTeX code for equation
```

### Tip 2: Simplify complex expressions
```python
expr_simple = sp.simplify(complicated_expr)
expr_trig = sp.trigsimp(complicated_expr)
```

### Tip 3: Fast evaluation with lambdify
```python
from sympy.utilities.lambdify import lambdify
N_r_func = lambdify((model.M, model.p, model.a_var, ...), N_r, 'numpy')
# Now: N_r_func(M, p, a, ...) is 100–1000× faster than .subs()
```

### Tip 4: Numerical solving
```python
from scipy.optimize import fsolve
solution = fsolve(residual_function, x0=initial_guess)
```

### Tip 5: Export to C code
```python
from sympy.printing.ccode import ccode
print(ccode(phi_expr))  # → C code for your embedded system
```

## 🐛 Common Errors & Fixes

### Error: "cannot import"
**Fix:** Make sure `motorcycle_dynamics_solver.py` is in same directory or Python path

### Error: "float object has no attribute 'subs'"
**Fix:** Don't forget `.evalf()` at the end; use `.subs()` on the SymPy expression, not the float

### Error: "takes 2 positional arguments but 3 were given"
**Fix:** In lambdify, pass symbols as tuple: `(a, b, c)` not `a, b, c`

### Performance: Very slow
**Fix:** You're probably using `.subs()` in a loop. Use `lambdify()` instead!
